# WAS-CORE-020 — Glossary

---

# Executive Summary

Glossary เป็นศูนย์กลางของคำศัพท์มาตรฐานที่ใช้ภายใน Web Apps Studio OS

ทุกเอกสารใน Repository ควรใช้คำจำกัดความจากเอกสารนี้ เพื่อลดความคลาดเคลื่อนในการสื่อสารและทำให้ผู้เชี่ยวชาญทุกบทบาท รวมถึง AI ใช้ภาษาที่สอดคล้องกัน

---

# Purpose

* กำหนดคำศัพท์มาตรฐาน
* ลดความกำกวมของคำ
* ใช้เป็นแหล่งอ้างอิงร่วมกัน
* สนับสนุนการทำงานร่วมกันของทุก Expert

---

# Scope

เอกสารนี้ครอบคลุมคำศัพท์ที่ใช้ใน

* Product Management
* Business Analysis
* Solution Architecture
* Engineering
* QA
* DevOps
* Documentation

---

# Standard Terminology

| Term                       | Definition                                            |
| -------------------------- | ----------------------------------------------------- |
| Product                    | ผลิตภัณฑ์ที่พัฒนาเพื่อตอบโจทย์ผู้ใช้งานและธุรกิจ      |
| Project                    | ชุดของงานที่ดำเนินการเพื่อสร้างหรือปรับปรุง Product   |
| Requirement                | ความต้องการที่ระบบต้องรองรับ                          |
| Functional Requirement     | สิ่งที่ระบบต้องสามารถทำได้                            |
| Non-functional Requirement | คุณสมบัติด้านคุณภาพของระบบ เช่น Performance, Security |
| User Story                 | ความต้องการของผู้ใช้ในรูปแบบ As a / I want / So that  |
| Acceptance Criteria        | เงื่อนไขที่ใช้ตรวจสอบว่างานเสร็จสมบูรณ์               |
| MVP                        | Minimum Viable Product สำหรับการเปิดใช้งานเวอร์ชันแรก |
| Stakeholder                | ผู้มีส่วนได้ส่วนเสียกับผลิตภัณฑ์                      |
| Architecture               | โครงสร้างของระบบทั้งในเชิงเทคนิคและองค์ประกอบ         |
| Standard                   | กฎหรือแนวปฏิบัติที่ทุกคนต้องปฏิบัติตาม                |
| Template                   | เอกสารต้นแบบที่นำกลับมาใช้ซ้ำได้                      |
| Checklist                  | รายการตรวจสอบก่อนส่งมอบ                               |
| Playbook                   | ขั้นตอนปฏิบัติมาตรฐานสำหรับกระบวนการหนึ่ง             |
| Handoff                    | การส่งมอบงานระหว่างบทบาท                              |
| Deliverable                | ผลลัพธ์ที่ต้องส่งมอบ                                  |
| Risk                       | ความเสี่ยงที่อาจส่งผลต่อโครงการ                       |
| Assumption                 | ข้อสมมติที่ยังไม่ได้รับการยืนยัน                      |
| Constraint                 | ข้อจำกัดของโครงการ                                    |
| Dependency                 | สิ่งที่ต้องพึ่งพาเพื่อให้ดำเนินงานต่อได้              |

# Usage Rules

* ใช้คำศัพท์จากเอกสารนี้เป็นหลัก
* หากมีคำใหม่ ให้เพิ่มลงใน Glossary ก่อนใช้งานในเอกสารอื่น
* หลีกเลี่ยงการใช้คำเดียวกันแต่มีหลายความหมาย

---

# Best Practices

* ใช้คำศัพท์ให้สอดคล้องกันทุกเอกสาร
* อ้างอิง Glossary เมื่อสร้าง Template ใหม่
* ตรวจสอบคำศัพท์ใหม่ในขั้นตอน Review
